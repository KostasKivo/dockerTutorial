Kubernetes Basics

We use kubectl always on our local machine to send instructions to the Kubernetes cluster

kubectl


We create objects with the kubectl create command. A deployment, deployes and manages nodes. Kubernetes works with objects. The deployment we create goes directly to the Kubernetes cluster

kubectl create deployment appName --image=imageNameOnDockerHub


See all deployments you have

kubectl get deployments


See all pods you have

kubectl get pods


Delete deployment

kubectl delete deployment appName


When we create a deployment the command is run by Master Node's Control plane. Which is send to the scheduler to find the best Node to create the Pod(s) Once the Worker is found kubelet manages and creates what it's required And then a new Pod is created with the container of our Image inside of it


A service object exposes pods to the cluster or externally. All pods inside a service have the same IP and allow external acess to Pods


Expose a delpoyment object with appName and we add a LoadBalancer type

kubectl expose deployment appName --type=LoadBalancer --port=8080


Scale the deployment object of the first app with 3 pods

kubectl scale deployment/first-app --replicas=3


Set new image for deployment

kubectl set image deployment/appName containerName=imageNameOnDockerHub:tagName


Undo latest deployment

kubectl rollout undo deployment/appName


In imperative declaration we always create the objects we want. In Declarative we point to a yaml file (similar to docker-compose)


Declarative pointing to a yaml file

kubectl apply -f=pathToFile.yaml


The selector inside a yaml files points to which pods to controll with their labels. It will only control the specified labels pods


In Kubernetes different volume types store your data in different ways. Also volumes are attached to pods


emptyDir is just a directory inside the Pod. hostPath is a bind mount between host and pod(s). csi volume is flexible and allows any volume type to be attached


Persistent volumes are independent from the nodes and pods and are inside the architecture of the cluster and the need to be claimed by a PVC


Services give us stable ip adresses (because the ip of nodes change) and also services allow outside world access
